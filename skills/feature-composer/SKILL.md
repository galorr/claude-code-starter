# FeatureComposer — PRD to Jira Tickets

Transforms a PRD + prototype + empty Jira epic into a fully ticketed implementation plan.

## Required Inputs
- `prdLink` — Google Docs PRD URL
- `prototypeUrl` — Preview/prototype URL
- `epicKey` — Empty Jira epic key (e.g. `PROJ-12345`)

**If any input is missing → stop and request it before proceeding.**

## Workflow

### Step 1: Extract PRD Content
Fetch the PRD using WebFetch.
Store as `prdContent`.

### Step 2: Analyze Prototype
Browse prototype URL using WebFetch. Capture:
- Key UI flows and pages
- Features and interactions visible
- Navigation structure

Store as `protoNotes`.

### Step 3: Enrich the Jira Epic

Build epic description with these sections:

```
Overview
[What the feature is, why it matters, strategic fit]
PRD Document: [prdLink]

Location in App
[Screen/page affected, where feature appears, access conditions]

Design References
[Figma links, before/after screenshots if available]

Design Change Detection
[Compare before/after to identify: visual changes, layout differences,
text/icons/buttons, styling adjustments. Flag TODOs if unclear.]

Functional Requirements
[Scope P1/P2, UI behavior, API/backend logic, constraints]

Test Cases
[Key flows, expected results, edge cases, permissions]

Dependencies
[APIs, systems, sequence dependencies]

Notes
[Additional context]
```

Update epic via Jira MCP.

### Step 4: Propose Ticket Breakdown

Based on PRD + prototype, suggest task categories. Typical breakdown:
- Frontend implementation + feature flag/release strategy
- Backend/API work
- Automation tests
- Analytics/tracking

**Do NOT add tickets for minor or passive changes.**

Present list to user → wait for approval or edits.

### Step 5: User Approval Loop
- Wait for approval
- If not approved → refine and re-present
- Once approved → add each to todo list, proceed to creation

### Step 6: Create Each Ticket (One by One)

For each approved task, generate a Jira story using this structure:

```markdown
## User Story
As a [persona], I want [feature] so that [outcome].

## Goal & Scope
[Brief purpose and importance of this specific task]

## UI/UX Behavior
- Layout & Navigation
- Key Components
- User Interactions
[Only include what's directly relevant to THIS ticket]

## Functional Requirements
1. [Numbered, high-level requirements]

## Components & Logic
- [Angular components, services, utilities for this task only]

## Acceptance Criteria
- Given/When/Then (5-8 statements, scoped to this ticket)

## Reference Links
- PRD: [prdLink]
- Prototype: [prototypeUrl]

Generated by FeatureComposer
```

**Formatting rules:**
- No emojis or asterisks in headers/body
- Use `##` headers, `-` for bullets, `1.` for numbered lists
- One blank line between sections
- No double spaces, no unsupported Markdown
- Include ONLY information relevant to this specific ticket

### Step 7: Wrap Up

After all stories created:
- Display summary of all tickets with links
- Confirm epic is fully decomposed and implementation-ready

## Important Reminders
- Every story must link to epic via `customfield_10008`
- Tailor each ticket's content to its scope only — don't copy/paste from epic
- Never invent styling details (colors, gradients, spacing) not in the PRD
- No markdown files as output — all goes into Jira
