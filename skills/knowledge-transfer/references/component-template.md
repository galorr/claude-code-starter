# Component Documentation Template

Markdown template for per-unit documentation files generated by the knowledge-transfer skill. Each analyzable unit (service, controller, module, etc.) gets its own file following this structure.

## Template

Use this structure when generating `knowledge-transfer/components/<unit-slug>.md`. Omit sections that don't apply to the unit (e.g., skip "API Endpoints" for a utility, skip "Database Entities" for a controller with no direct DB access).

---

### File Header

```markdown
# <Unit Name>

**Type:** <Service | Controller | Module | Component | Utility | Model | Middleware | ...>
**Location:** `<relative/path/to/main/file>`
**Last modified:** <date from git log>

<One or two sentences describing what this unit does and why it exists.>
```

## Slugging Convention

The filename slug is derived from the unit name:
- Kebab-case: `UsersService` → `users-service`
- Preserve meaningful prefixes: `AdminUsersController` → `admin-users-controller`
- Cross-cutting docs get an underscore prefix: `_architecture-overview`, `_dependency-graph`
