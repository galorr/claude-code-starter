## Guidelines

- **Be factual, not prescriptive.** Document what the code does, not what it should do. If a pattern is inconsistent, note both the majority approach and the exceptions.
- **Minimum threshold:** Only document patterns observed in at least 3 units or 20% of relevant units (whichever is higher). Don't list one-off patterns as conventions.
- **Code snippets:** Keep examples short (3-5 lines max). Always attribute with file path.
- **Exceptions are valuable.** Deviations from patterns often indicate tech debt, intentional design decisions, or areas that need attention. Note them without judgment.
