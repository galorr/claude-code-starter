# HTML Navigator Template

Implementation guidance for the self-contained HTML navigator generated by the knowledge-transfer skill. The navigator is a single `index.html` file with no external dependencies that works via `file://` protocol.

## Architecture

The HTML file contains three parts:

1. **HTML structure** — semantic layout with sidebar, toolbar, and content area
2. **CSS** — embedded `<style>` block with light/dark themes
3. **JavaScript** — embedded `<script>` blocks for navigation, search, rendering, and data

The generated markdown content is embedded as a JSON array in a `<script type="application/json" id="docs-data">` block.

## Data Format

```html
<script type="application/json" id="docs-meta">
{
  "repo": "owner/repo",
  "repoUrl": "https://github.com/owner/repo/blob/<commitSha>/",
  "commitSha": "<commitSha>",
  "generatedAt": "<ISO timestamp>",
  "framework": "<detected framework>"
}
</script>
<script type="application/json" id="coverage-data">
{
  "totalSourceFiles": 156,
  "documentedFiles": 142,
  "documentedPct": 91.0,
  "withGitHubContextPct": 80.8,
  "withJiraContextPct": 48.9,
  "withDiagramsPct": 25.5,
  "staleFiles": 0
}
</script>
<script type="application/json" id="docs-data">
[
  {
    "slug": "users-service",
    "title": "Users Service",
    "category": "Service",
    "content": "# Users Service\n\n**Type:** Service\n..."
  },
  {
    "slug": "_architecture-overview",
    "title": "Architecture Overview",
    "category": "Overview",
    "content": "# Architecture Overview\n\n..."
  }
]
</script>
<script type="application/json" id="graph-data">
{
  "nodes": [
    { "id": "users-service", "label": "Users Service", "category": "Service" },
    { "id": "users-controller", "label": "Users Controller", "category": "Controller" }
  ],
  "edges": [
    { "source": "users-controller", "target": "users-service", "type": "injects" }
  ]
}
</script>
<script type="application/json" id="learning-paths-data">
[
  {
    "title": "Backend Onboarding",
    "description": "For engineers joining the backend team",
    "steps": [
      { "slug": "_architecture-overview", "note": "Start with the big picture" },
      { "slug": "app-module", "note": "The application bootstrap and module wiring" },
      { "slug": "users-service", "note": "Core business logic example" }
    ]
  }
]
</script>
```

**Metadata fields (`docs-meta`):**
- `repo` — owner/repo identifier
- `repoUrl` — base URL for source links (includes commit SHA for permalinks)
- `commitSha` — the git commit SHA at generation time
- `generatedAt` — ISO timestamp of generation
- `framework` — detected framework name

**Graph fields (`graph-data`):**
- `nodes[].id` — unit slug, matches docs-data slug for cross-navigation
- `nodes[].label` — display name for graph node
- `nodes[].category` — unit type, used for color-coding
- `edges[].source` — slug of the source node (depends on target)
- `edges[].target` — slug of the target node (depended upon)
- `edges[].type` — relationship type (imports, injects, extends, etc.)

**Learning paths fields (`learning-paths-data`):**
- `[].title` — path name (e.g., "Backend Onboarding")
- `[].description` — one-sentence audience description
- `[].steps[].slug` — doc slug to navigate to
- `[].steps[].note` — brief reason why this doc is in the sequence

**Document fields (`docs-data`):**
- `slug` — filename without `.md` extension, used for URL hash routing
- `title` — human-readable display name for the sidebar
- `category` — unit type (Service, Controller, Module, Overview, etc.)
- `content` — raw markdown content of the file
- `startHere` — (optional) `true` for the 5-10 most critical docs tagged in Phase 4

**Ordering:** Cross-cutting docs (slugs starting with `_`) appear first, sorted alphabetically. Then per-unit docs sorted alphabetically within each category.

### Layout

- Sidebar: fixed left, `var(--sidebar-width)` wide, full height, scrollable
- Content: fills remaining width, centered reading column (max-width ~800px), padded
- On mobile (< 768px): sidebar overlays content, toggled by hamburger button

### Typography

- Body text: 16px, line-height 1.6, `var(--font-sans)`
- Code blocks: 14px, `var(--font-mono)`, `var(--code-bg)` background, rounded corners, horizontal scroll
- Headings: bold, appropriate scale (h1: 1.8em, h2: 1.4em, h3: 1.2em)
- Tables: full-width, bordered, alternating row backgrounds, horizontal scroll wrapper

### Coverage Bar

- Displayed at the top of the content area when no doc is selected (initial view / dashboard)
- Shows 4 stat cards in a horizontal row: "Files Documented", "GitHub Context", "Jira Context", "Diagrams"
- Each card shows a percentage value with a small horizontal progress bar underneath
- If `staleFiles > 0`, show a warning badge: "N files changed since last run"
- Cards use `var(--bg-secondary)` background, rounded corners, subtle shadow
- On mobile: cards stack in 2x2 grid

### View Toggle

- Positioned above the main content area (between sidebar and content)
- Two buttons: "Docs" (default active) and "Dependency Graph"
- Active button has accent background, inactive has secondary background
- Clicking "Dependency Graph" hides `<main>` and shows `<div class="graph-view">`
- Clicking "Docs" reverses the toggle

### Graph View

- Full-width, full-height container (fills the space where content would be)
- SVG element fills the container
- Nodes: colored circles (radius ~20px) with category-based colors matching filter chip colors
- Node labels: text below each circle, truncated if > 15 chars
- Edges: lines with small arrowheads at the target end, color based on edge type
- Hover state: hovered node and all connected edges/nodes highlighted, rest dimmed (opacity 0.3)
- Pan: click-and-drag on background translates the SVG viewBox
- Zoom: mouse wheel scales the SVG viewBox
- Legend: small legend in bottom-right showing category colors

### Filter Chips

- Small rounded pills, one per category
- Click to toggle: active = accent background, inactive = secondary background
- All active by default

### Learning Paths Panel

- Toggled via a "Learning Paths" button in the sidebar header (next to the filter chips area)
- When active, replaces the category-filtered nav list with the learning paths view:
  - Collapsible path sections, each with a title header and description
  - Ordered numbered list of doc links within each path
  - Each link shows the doc title + the brief note in muted text
  - Clicking a link navigates to that doc and highlights the current step
- When toggled off, returns to the normal category-filtered navigation

### "Start Here" Badges

- Docs with `startHere: true` in their data show a small accent-colored pill badge labeled "Start" next to their title in the sidebar navigation
- Badge uses `var(--accent)` background with white text, font-size 10px, rounded corners
- Visible in both normal nav view and learning paths view

### Sidebar Navigation

- Grouped by category with category headers
- Active item highlighted with accent color left border
- Hover effect on items
- Items show title only, not slug

## JavaScript Requirements

### Markdown Renderer

A minimal inline markdown-to-HTML renderer (~50 lines). Must support:

- **Headings**: `# h1` through `###### h6`
- **Bold**: `**text**`
- **Italic**: `*text*`
- **Code blocks**: triple-backtick fenced blocks (with optional language hint, ignored for rendering)
- **Inline code**: `` `code` ``
- **Tables**: pipe-delimited tables with header separator row
- **Unordered lists**: `- item` and `* item` (including nested)
- **Ordered lists**: `1. item`
- **Links**: `[text](url)`
- **Horizontal rules**: `---`
- **Line breaks**: double newline → paragraph

Process in this order to avoid conflicts:
1. Code blocks (protect from further processing)
2. Tables
3. Headings
4. Horizontal rules
5. Lists
6. Inline formatting (bold, italic, code, links)
7. Paragraphs

### Search

- Bind to the search input's `input` event
- Filter sidebar items by matching query against title and raw markdown content (case-insensitive)
- Debounce at 150ms for performance
- Show/hide sidebar items based on match
- If the currently viewed doc is filtered out, don't change it — only the sidebar list updates

### Category Filters

- Extract unique categories from the data
- Render a chip per category
- Clicking a chip toggles that category's visibility in the sidebar
- Multiple categories can be active simultaneously
- Active state stored in a Set, re-render sidebar on change

### Dependency Graph Renderer

A minimal force-directed graph layout rendered to SVG (~100 lines JS). No external libraries required.

**Layout algorithm:**
1. Initialize node positions randomly within the container bounds
2. Run a spring-force simulation (~50 iterations):
   - Repulsion between all node pairs (Coulomb's law, ~100px equilibrium)
   - Attraction along edges (Hooke's law, rest length ~150px)
   - Damping factor 0.9 per iteration
3. After simulation, normalize positions to fit within the SVG viewBox

**Rendering:**
1. Create SVG element with viewBox matching container dimensions
2. Render edges as `<line>` elements with arrow markers (`<defs><marker>`)
3. Render nodes as `<circle>` elements with category-based fill colors
4. Render labels as `<text>` elements below each node
5. Apply CSS transitions for hover effects

**Interactions:**
- **Click node**: set `location.hash` to the node's slug, switch view to "Docs", navigate to that doc
- **Hover node**: add `.highlighted` class to the node and all connected edges/nodes. Add `.dimmed` class to all others
- **Pan**: on mousedown on SVG background, track mousemove to translate viewBox origin
- **Zoom**: on wheel event, scale viewBox dimensions (min 0.5x, max 3x)

**Color mapping (same palette as category filter chips):**
- Service → blue, Controller → green, Module → purple, Component → orange, Model → red, Utility → gray, Overview → teal

### URL Hash Routing

- On load: read `location.hash`, navigate to that doc (or first doc if no hash)
- On sidebar item click: set `location.hash` and render the doc
- Listen for `hashchange` event to handle browser back/forward

### Theme Toggle

- Check `localStorage.getItem('kt-theme')` on load
- If no stored preference, check `prefers-color-scheme: dark` media query
- Toggle sets `data-theme` attribute on `<html>` and saves to `localStorage`
- Button icon switches between sun (in dark mode) and moon (in light mode)

### Initialization

```javascript
document.addEventListener('DOMContentLoaded', () => {
  const meta = JSON.parse(document.getElementById('docs-meta').textContent);
  const coverage = JSON.parse(document.getElementById('coverage-data').textContent);
  const docs = JSON.parse(document.getElementById('docs-data').textContent);
  const graphData = JSON.parse(document.getElementById('graph-data').textContent);
  const learningPaths = JSON.parse(document.getElementById('learning-paths-data').textContent);
  initTheme();
  renderCoverageBar(coverage);
  renderFilters(docs);
  renderNav(docs);
  renderLearningPaths(learningPaths, docs);
  initGraphView(graphData, docs);
  bindViewToggle();
  bindLearningPathsToggle();
  navigateFromHash(docs);
  bindSearch(docs);
  bindHashChange(docs);
});
```

## Generation Process

When building `index.html` in Phase 5:

1. Read all `.md` files from `knowledge-transfer/components/`
2. For each file:
   - Extract slug from filename (strip `.md`)
   - Extract title from first `# ` heading
   - Determine category from the `**Type:**` line, or `Overview` for `_`-prefixed files
3. Sort: overview docs first (alphabetical), then per-unit docs grouped by category (alphabetical within each)
4. Build the JSON data array for docs-data
5. Build the graph-data JSON from Phase 5.5 knowledge-graph.json (simplified to just id/label/category for nodes)
6. Build the coverage-data JSON from the coverage metrics computed during Phase 3.5
7. Build the docs-meta JSON from Phase 0 values (repo, commitSha, framework, timestamp)
8. Escape all JSON blocks for safe embedding: `JSON.stringify(data).replace(/<\/script>/gi, '<\\/script>')`
9. Assemble the full HTML string and write to `knowledge-transfer/index.html`

After generating `index.html`, verify:
- [ ] File opens in browser via `file://` protocol without errors
- [ ] All docs appear in sidebar
- [ ] Search filters sidebar items correctly
- [ ] Category chips toggle visibility
- [ ] Dark/light mode toggle works
- [ ] Hash routing works (direct URL and back/forward)
- [ ] Code blocks render with monospace font
- [ ] Tables render with borders and alignment
- [ ] Mobile layout works (sidebar collapses)
- [ ] Coverage bar displays metrics with progress bars
- [ ] Graph view renders all units as nodes with edges
- [ ] Clicking a node in graph view navigates to that doc (switches to Docs view)
- [ ] Graph pan and zoom work
- [ ] Node hover highlights connected edges and nodes
- [ ] Learning Paths toggle shows/hides the paths panel in the sidebar
- [ ] Clicking a learning path step navigates to that doc
- [ ] "Start here" badges appear on critical docs in the sidebar
